## Доклад

### Зачем нужна персональная страница для вашего проекта?

Целью этого доклада является показать как можно быстро и просто сделать лендинг для вашего проекта. Не важно,
коммерческий это проект или какое-то Open Source решение. Лендинг – это инструмент показать проект во всей красе,
подсветить его сильные стороны и заинтересовать потенциального пользователя.

Т.к. я являюсь архитектором, который в прошлой жизни был разработчиком, мне нравится программировать и я иногда пишу
какие-то проекты для обучения или коммерческого использования. И в этих проектах часто появлялась задача маскировать
sensitive данные в логах.

Под sensitive данными мы подразумеваем, в том числе персональные данные:

> Федеральный закон № 152-ФЗ «О персональных данных» регулирует обработку персональных данных физических лиц и
> направлен на защиту их прав и свобод, в частности права на неприкосновенность частной жизни.
> Кратко: закон определяет, как можно собирать, хранить, использовать и передавать персональные данные, и вводит
> ограничения для операторов данных (компаний, сервисов, госорганов).
> Запрещено:
> – хранить данные без необходимых мер безопасности;
> – допускать утечки, несанкционированный доступ.

Отсюда следует, что если у вас в логах появляются ФИО + email, то это уже персональные данные, которые вы не можете
отправлять в какую-то облачную систему агрегации логов. Следовательно, нужно эти данные маскировать. Чтобы не
реализовывать эту функциональность одинаково в нескольких проектах, я вынес код в библиотеку и опубликовал на
Github ([log4j2-masking-lib](https://github.com/Romanow/log4j2-masking-lib)).

На мой взгляд реализация получилась неплохой и я задумался над тем, чтобы это решение могли использовать другие.

Вначале проект выглядел вот так:

![Without README.md](images/Without%20README.md.png)

Как вы понимаете, шанс что кто-то захочет использовать это решение равнялся примерно нулю. Нужно было:

* описать решение;
* показать как настраивать и кастомизировать;
* оформить это так, чтобы было приятно с этим работать.

### Делаем простейшую лендинг из `README.md` проекта

Первое, что нужно сделать – это описать `README.md` в проекте, где рассказать про то как подключать библиотеку,
кастомизировать и описать внутреннее устройство.

Получилось вот так:

![With README.md](images/With%20README.md.png)

Согласитесь, получилось лучше: вы сразу видите как подключить и настроить библиотеку.

Т.к. мы работаем в экосистеме Github, мы можем легким росчерком руки `README.md` опубликовать как статическую страницу
через Github Pages (кстати, в Gitverse тоже появились pages) и даже подключить свой домен:

![Publish with Github Pages](images/Publish%20with%20Github%20Pages.png)

![log4j2-masking-lib landing page](images/log4j2-masking-lib%20landing%20page.png)

Описывать проект весьма унылая задача, поэтому можно ее делегировать какому-нибудь GPT, а потом просто проверить его
результат.

### А теперь делаем красивую страницу

На этом этапе у нас уже есть базовый лендинг вашего проекта. Теперь давайте сделаем его красивым.

Здесь давайте немного разомнемся: кто только недавно вошел в IT?

Сколько бы времени делали небольшой лендинг для этого проекта? День / два?

Давайте это сделаем за 10 минут?

```text
Мне нужно сделать лендинг для старицы проекта log4j2-masking-lib описывающий проект. Стиль формальный, описание возьми из README.md.

Структура страницы:
– Заголовок: Open Source, название проекта, описание, ссылка на GitHub и на Latest Release, а так же список бейджей из README.md.
– Тело сайта раздели на строки по 1 или 2 колонки:
    – На первой строке "Назначение".
    – Дальше идет две колонки: "Подключение" и "Конфигурация правил".
    – Далее строка "Интеграция с Log4j2".
    – Потом "Правила маскирования" в виде таблицы (добавь примеров из тестов).
– Footer – ссылка на автора (Романов Алексей: https://github.com/Romanow, https://romanow-alex.ru/) и на лицензию.

CSS выноси в отдельный файл. Для CSS можешь использовать библиотеку MUI, либо сделать свой CSS под Material UI.
Результат положи в папку landing.
```

Так же вы можете создать репозиторий под названием `<owner>.github.io` и там создать свою домашнюю страничку, так же
опубликовать ее как Github Pages и подключить к ней свой домен.
